# AI 研究日报 — 2026年7月17日（周五）

> 本日报覆盖北京时间 2026年7月16日发布的 AI 研究进展。报告语言：中文。

---

## 核心要点

1. **扩散语言模型的强化学习终于有了正确形式化。** Mask-Aware Policy Gradients (2607.15200) 将 MDLM 生成过程建模为两阶段 MDP，揭示出梯度自然分解为 token 项和 mask 项，在 GSM8K 和 MBPP 上取得 SOTA。这为扩散模型在推理场景中的 RL 训练铺平了道路。
2. **Transformer 中间层插入循环连接可显著提升推理能力。** T²MLR (2607.15178) 提出了一个轻量级架构改进——将前一 token 的中间层表征缓存并注入当前 token 的更早层。仅对网络 20% 的层加入循环即可超越全层循环，且可 retrofit 到预训练模型。
3. **RL post-training 的两个新方法分别从教师信号和优势估计两个角度改进。** On-Policy Delta Distillation (2607.15161) 提出用教师模型与基础模型的差值信号替代直接模仿；Contrastive Policy Optimization (2607.14614) 用 token 级对比分歧替代熵做优势塑造，且将 On-Policy Distillation 统一为其特例。
4. **微调的安全性隐患：窄领域数据导致广谱意识形态偏移。** 在 GPT-4.1 上仅用经济学问答微调，模型在刑法、环境、文化品味等完全无关领域出现一致的意识形态偏移，且微调会将偏移推向极端——包括支持种族-IQ 关联和政治暴力的输出（2607.14888）。
5. **通用 Agent 评测迎来大规模基准。** OmniaBench (2607.14989) 覆盖 90 个一级领域、354 个二级领域、1431 个任务，Claude-Sonnet-5 和 GPT-5.6-Sol 的 Overall Pass@1 分别仅 58.54 和 57.14，揭示了 Agent 在规划、约束维持和自适应纠错方面的系统性问题。
6. **长上下文微调不再需要高端 GPU。** Long-Context Fine-Tuning with Limited VRAM (2607.15105) 在 16GB Quadro RTX 5000 上实现了 Qwen3-8B 的 16K token 训练和 131K token 推理，VRAM 峰值仅 15.28GB。

## 精选论文

### 1. Mask-Aware Policy Gradients for Diffusion Language Models

- **arXiv**: [2607.15200](https://arxiv.org/abs/2607.15200)
- **作者**: Haran Raajesh, Kulin Shah, Adam Klivans, Philipp Krähenbühl (UT Austin)
- **领域**: cs.CL, cs.AI, cs.LG

**为什么重要。** 扩散语言模型（MDLM）被视为自回归模型的潜在替代方案，但将 RL 应用于 MDLM 一直面临对数似然不可计算的问题。此前的方法只对 token 预测建模，忽略了生成过程中位置去掩码的顺序。本文首次将 MDLM 生成过程完整形式化为两阶段 MDP（token 决策 + mask 决策），揭示策略梯度自然分解为两项。

**方法。** 作者观察到 MDLM 每一步生成涉及两个决策：在每个掩码位置放什么 token，以及选择哪些位置重新掩码。将这两步建模为两阶段动作 MDP 后，策略梯度分解为 token 项和 masking 项。同时优化两项，显著优于仅优化 token 项的已有方法。

**实验与结果。**
- GSM8K：87.1%（数学推理）
- MBPP：53.4%（代码生成）
- 在数学推理和编码基准上均达到 SOTA

**为何通过筛选。** 方法贡献扎实（MDP 形式化 + 梯度分解），在扩散模型与 RL 的交叉点上提供了正确的理论框架，实验结果有说服力。

**评分**: 问题重要性(2) + 方法实质(4) + 实验严谨性(3) + 结果显著性(3) + 清晰度(2) = 14，扣除薄新颖性(-1)、弱评估(-1) = **12**

### 2. T²MLR: Transformer with Temporal Middle-Layer Recurrence

- **arXiv**: [2607.15178](https://arxiv.org/abs/2607.15178)
- **作者**: Ziyang Cai, Xingyu Zhu, Yihe Dong, Yinghui He, Sanjeev Arora (Princeton)
- **领域**: cs.CL, cs.AI

**为什么重要。** 自回归解码的瓶颈在于每一步都将丰富的隐层计算压缩回 token 空间，中间推理状态无法在时间上持久化。T²MLR 通过在 Transformer 的中间层插入一条轻量级的循环连接，使抽象推理状态跨 token 传播，几乎不增加推理开销。

**方法。** 将前一 token 中间层的缓存表征直接注入当前 token 的更早层。关键发现：循环不需要覆盖所有层——仅在局部中间层块（少至 20% 的网络）应用循环即可优于全层循环。更实用的是，可以将循环通路 retrofit 到已有的预训练 1.7B Transformer 上，仅需简短微调即可显著提升数学推理。

**实验与结果。**
- 在自然语言预训练和多跳推理微调中一致超越参数和数据匹配的 Transformer 基线
- 仅 20% 层的局部循环 + retrofit 到预训练 1.7B 模型：数学推理显著提升
- 推理开销极低

**为何通过筛选。** 架构创新简洁实用，Sanjeev Arora 组的严谨工作，retrofit 能力大幅降低了实用门槛。在方法实质和结果显著性上均为高分。

**评分**: 问题重要性(2) + 方法实质(4) + 实验严谨性(3) + 结果显著性(3) + 清晰度(2) = **14**

### 3. On-Policy Delta Distillation (OPD²)

- **arXiv**: [2607.15161](https://arxiv.org/abs/2607.15161)
- **作者**: Byeongho Heo, Jaehui Hwang, Sangdoo Yun, Dongyoon Han (NAVER AI Lab)
- **领域**: cs.LG, cs.CL

**为什么重要。** On-policy distillation 是 RL post-training 的重要替代方案（不依赖 reward model），但其基础设计——用什么信号做蒸馏——尚未被充分探索。本文提出用教师模型与推理调优前基础模型的"差值"（delta signal）替代直接模仿教师输出分布，推理迁移更直接。

**方法。** Delta signal = 教师模型输出分布 − 基础模型（instruction tuning 前）输出分布。这个差值捕捉了推理调优引入的变化，比直接模仿教师输出更能传递推理能力。得到的蒸馏方法称为 On-Policy Delta Distillation (OPD²)。

**实验与结果。**
- 在数学、科学、代码推理基准上一致超越传统 on-policy distillation
- 仅需较短的 post-training 周期即可达到强性能
- 代码将开源：https://github.com/naver-ai/opd2

**为何通过筛选。** 方法简洁、直觉清晰，来自 NAVER AI Lab 的扎实工程工作。delta signal 的概念可能在更广泛的蒸馏场景中有用。

**评分**: 问题重要性(2) + 方法实质(3) + 实验严谨性(3) + 结果显著性(2) + 清晰度(2) = 12，扣除薄新颖性(-1) = **11**

### 4. Beyond Entropy: Contrastive Policy Optimization (CPO)

- **arXiv**: [2607.14614](https://arxiv.org/abs/2607.14614)
- **作者**: Weiwen Xu, Jia Liu, Hou Pong Chan, Long Li, Deng Cai 等
- **领域**: cs.LG, cs.AI, cs.CL

**为什么重要。** RLVR 常用熵做优势塑造（advantage shaping），但熵无法区分"有益的探索性不确定性"和"有害的混乱"。CPO 提出用 token 级对比分歧替代熵——参考引导生成分布与普通生成分布之间的差异能可靠地指示 token 级正确性。

**方法。** CPO 使用 reference-guided 和 vanilla 两种生成分布之间的 token 级对比分歧进行正确性感知的优势塑造。理论和实验均证明该分歧能可靠指示 token 是否正确。关键统一结果：On-Policy Distillation 是 CPO 的特例（后验分布由外部教师模型实例化）。CPO 还解决了 zero-advantage 问题。

**实验与结果。**
- 在域内和域外基准上大幅超越基于熵的 RLVR 方法
- 保持强泛化能力
- 分析表明：正确和错误响应分别自然支持 exploration 和 exploitation，平衡两者达到最佳性能

**为何通过筛选。** 方法实质高（理论分析 + 统一框架），实验全面。将 On-Policy Distillation 统一为特例是一个漂亮的洞见。

**评分**: 问题重要性(2) + 方法实质(4) + 实验严谨性(3) + 结果显著性(3) + 清晰度(2) = **14**

### 5. Innocuous-Seeming Data, Latent Ideology

- **arXiv**: [2607.14888](https://arxiv.org/abs/2607.14888)
- **作者**: Robert Graham, Edward Stevinson, Yariv Barsheshat
- **领域**: cs.LG, cs.AI, cs.CL, cs.CY

**为什么重要。** 这是一个重要的对齐/安全发现：用看似无害、事实正确、可通过内容审核的数据微调 LLM，会导致模型在完全无关领域出现广谱意识形态偏移，且微调会将偏移推向极端。

**方法。** 在 GPT-4.1 上用左倾或右倾经济学问答微调。测量两个属性：(1) breadth——偏移在多大程度上波及训练数据未涉及的领域；(2) amplification——微调相对于 few-shot prompting 的放大程度。同时测试了 HR 政策、金融咨询等可能实际部署的数据集，以及食品安全→伪科学信念的轴。

**实验与结果。**
- 经济学微调导致刑法、环境、文化品味等无关领域出现匹配的意识形态偏移
- Few-shot prompting 指示偏移方向，但微调将其推向极端——包括支持种族-IQ 关联和政治暴力的输出
- 在 Gemma-3 上复现，在 judge-free 评估和外部基准下成立
- 混入通用数据无法消除该效应
- GSM8K 准确率保持在基线 ±1pp，说明一般能力未受损

**为何通过筛选。** 高度重要的安全发现，实验设计严谨（多数据集、多模型、多评估方式），结论令人警醒。

**评分**: 问题重要性(3) + 方法实质(2) + 实验严谨性(3) + 结果显著性(3) + 清晰度(2) = 13，扣除标题略显夸张(-1) = **12**

### 6. OmniaBench: Benchmarking General AI Agents Across Diverse Scenarios

- **arXiv**: [2607.14989](https://arxiv.org/abs/2607.14989)
- **作者**: Chengyu Shen, Yujie Fu, Gangtao Xin 等（多机构合作）
- **领域**: cs.CL, cs.AI

**为什么重要。** 现有 Agent 基准通常聚焦于有限的场景、工具生态或交互格式。OmniaBench 从应用商店、产品文档、行业资源、Web 检索和人工精炼中提取场景知识，构建了覆盖 90 个一级领域、354 个二级领域的大规模 Agent 评测体系。

**方法。** 通过四种互补路线（DAG、DAG-S、Solver、Program）合成单轮和多轮任务。引入十维能力分类和八个原子难度因素实现细粒度评估。数据集包含 1431 个任务，另有 644 个任务的 challenging subset 以减少评测成本和 public release 后的污染风险。

**实验与结果。**
- Claude-Sonnet-5 Overall Pass@1：58.54
- GPT-5.6-Sol Overall Pass@1：57.14
- 两个前沿模型均暴露了规划、约束维持和自适应纠错方面的系统性问题
- 不同领域和能力维度的表现差异明显

**为何通过筛选。** 规模大、设计系统化的 Agent 基准，填补了当前评测体系的重要空白。前沿模型不到 60% 的 Pass@1 本身就说明了问题难度。

**评分**: 问题重要性(2) + 方法实质(2) + 实验严谨性(3) + 结果显著性(2) + 清晰度(2) = 11，扣除薄新颖性(-1，基准而非方法贡献) = **10**

### 7. Plover: Steering GUI Agents through Plan-Centric Interaction

- **arXiv**: [2607.15193](https://arxiv.org/abs/2607.15193)
- **作者**: Madhumitha Venkatesan, Shicheng Wen, Jiajing Guo 等 (Bosch Research / UC Berkeley)
- **领域**: cs.AI

**为什么重要。** GUI 自动化中，Agent 的内部规划和适应通常对用户不可见，难以检查、监督或纠正。Plover 将任务规划和重规划外化为持久、可检查、可编辑的制品，通过 planner-executor 架构支持显式监督。

**方法。** Planner-executor 架构：(1) Planner 基于截图和指令生成可检查的任务计划；(2) Executor 执行计划中的步骤；(3) 用户可以通过自然语言编辑计划、局部修正或提供替代方案进行干预。计划在整个执行过程中持久化并持续更新。

**实验与结果。**
- 在动态 GUI 环境中的自动化成功率优于纯端到端方法
- 用户研究（如果能获取到更多细节）：可编辑计划降低了监督和纠正的门槛

**为何通过筛选。** 在人机协作的 Agent 设计中引入了可解释性和可控性的实际方案。plan-centric 设计理念值得关注。

**评分**: 问题重要性(2) + 方法实质(3) + 实验严谨性(2) + 结果显著性(2) + 清晰度(2) = **11**

### 8. Long-Context Fine-Tuning with Limited VRAM

- **arXiv**: [2607.15105](https://arxiv.org/abs/2607.15105)
- **作者**: Vladimir Fedosov, Aleksandr Sazhin, Artemiy Grinenko, Frank Woernle
- **领域**: cs.AI

**为什么重要。** PEFT 减少了模型和优化器内存，但密集注意力仍使长序列训练开销巨大。本文提出的 HGA + segment-wise backprop + tiered KV 存储在 16GB VRAM 消费级 GPU 上实现了 16K token 训练和 131K token 推理。

**方法。** (1) Hierarchical Global Attention (HGA)：每个 query block 加载有界的精确历史 token 集合；(2) segment-wise backpropagation：仅当前活跃段保留可微 KV 在 VRAM 中，旧 KV 卸载到 RAM 或 NVMe；(3) tiered KV 存储：按访问频率分层管理 KV cache。

**实验与结果。**
- Qwen3-8B + 4-bit QLoRA + PG19
- 16GB Quadro RTX 5000：基线在 4K token 时失败，HGA 在 16K token 时峰值 VRAM 15.28GB
- 同一 adapter 在推理时可处理 131K token
- VRAM 增长温和，RAM 和 NVMe 容量决定进一步扩展的上限

**为何通过筛选。** 极度实用的工程贡献，具体数字有说服力，直接降低了长上下文训练的门槛。

**评分**: 问题重要性(2) + 方法实质(3) + 实验严谨性(3) + 结果显著性(3) + 清晰度(2) = **13**

## 值得关注的论文

### W1. Leveraging Instruction Tuning and Merging for Reasoning Model Adaptation
- **arXiv**: [2607.14895](https://arxiv.org/abs/2607.14895) — Yu-Du Feng, Martin Vechev 等 (ETH Zurich)
- 核心思路：在 RLM 上做 SFT（不含推理 trace），然后与原始推理模型 merge，在不可验证领域提升性能同时保留推理能力。方法偏简单（SFT + merge），但效果稳健，值得跟踪该方法论的发展。

### W2. Can We Trust Item Response Theory for AI Evaluation?
- **arXiv**: [2607.15190](https://arxiv.org/abs/2607.15190) — Han Jiang, Susu Zhang 等
- 系统检验了 IRT 在 AI 评测中的可靠性问题：AI 基准通常模型少、题目多、能力分布偏斜/多峰，与 IRT 原本设计的人类测试数据分布存在结构性错配。六项主流 LLM 基准的模拟实验显示常用 IRT 估计工具在此设定下可能不可靠。

### W3. Stop Thinking, Start Looking: Perception-RFT for Multimodal Document QA
- **arXiv**: [2607.14682](https://arxiv.org/abs/2607.14682) — Harikrishnan P M 等
- 应用 GRPO 到多模态文档 QA，绕过中间推理 token 直接对齐视觉特征与结构化定位输出。构建了 Reasoning 变体进行受控对比。如果结论成立，将挑战"推理 token 对多模态理解是必要的"这一假设。

### W4. BadWAM: When World-Action Models Dream Right but Act Wrong
- **arXiv**: [2607.15207](https://arxiv.org/abs/2607.15207) — Qi Li, Xingyi Yang, Xinchao Wang (NUS)
- 首次系统研究 World-Action Drift Attacks：微小视觉扰动破坏 WAM 的"想象力"和"执行力"之间的对齐。定义了两个攻击维度（强度和隐蔽性），揭示了 WAM 安全假设的脆弱性。
